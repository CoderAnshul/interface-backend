# JavaScript Fundamentals

## Introduction

JavaScript is a versatile, high-level programming language that is one of the core technologies of the World Wide Web, alongside HTML and CSS. It enables interactive web pages and is an essential part of web applications.

## What You'll Learn

In this comprehensive lesson, we'll cover:

• Variables and data types
• Functions and scope
• Objects and arrays
• Control structures
• Basic DOM manipulation

## Getting Started with Variables

JavaScript uses three keywords to declare variables:

**let** - for variables that can be reassigned
**const** - for constants that cannot be reassigned  
**var** - older syntax, function-scoped

### Examples:

let userName = "Alice";
const PI = 3.14159;
var age = 25;

## Data Types

JavaScript has several built-in data types:

### Primitive Types:
1. **Number** - Represents both integers and floating-point numbers
   Example: 42, 3.14, -17

2. **String** - Represents text data
   Example: "Hello World", 'JavaScript', `Template literal`

3. **Boolean** - Represents true/false values
   Example: true, false

4. **Undefined** - Variable declared but not assigned
   Example: let x; // x is undefined

5. **Null** - Intentional absence of value
   Example: let data = null;

6. **Symbol** - Unique identifiers (ES6+)
   Example: Symbol('id')

7. **BigInt** - Large integers (ES2020+)
   Example: 1234567890123456789012345678901234567890n

### Reference Types:
- **Object** - Collections of key-value pairs
- **Array** - Ordered lists of values
- **Function** - Reusable blocks of code

## Functions

Functions are first-class citizens in JavaScript, meaning they can be:
- Assigned to variables
- Passed as arguments
- Returned from other functions

### Function Declaration:
function greet(name) {
    return "Hello, " + name + "!";
}

### Function Expression:
const greet = function(name) {
    return "Hello, " + name + "!";
};

### Arrow Function (ES6+):
const greet = (name) => {
    return "Hello, " + name + "!";
};

// Or shorter:
const greet = name => "Hello, " + name + "!";

## Objects and Arrays

### Objects:
const person = {
    name: "John Doe",
    age: 30,
    city: "New York",
    greet: function() {
        return "Hello, I'm " + this.name;
    }
};

### Arrays:
const fruits = ["apple", "banana", "orange"];
const numbers = [1, 2, 3, 4, 5];
const mixed = ["text", 42, true, null];

## Control Structures

### Conditional Statements:

// if-else
if (age >= 18) {
    //console.log("You are an adult");
} else {
    //console.log("You are a minor");
}

// switch
switch (day) {
    case "Monday":
        //console.log("Start of work week");
        break;
    case "Friday":
        //console.log("TGIF!");
        break;
    default:
        //console.log("Regular day");

### Loops:

// for loop
for (let i = 0; i < 5; i++) {
    //console.log(i);
}

// while loop
let count = 0;
while (count < 5) {
    //console.log(count);
    count++;
}

// for...of (arrays)
for (const item of fruits) {
    //console.log(item);
}

// for...in (objects)
for (const key in person) {
    //console.log(key + ": " + person[key]);
}

## Modern JavaScript Features (ES6+)

### Template Literals:
const name = "World";
const greeting = `Hello, ${name}!`;

### Destructuring:
const [first, second] = fruits;
const {name, age} = person;

### Spread Operator:
const newArray = [...fruits, "grape"];
const newObject = {...person, country: "USA"};

### Default Parameters:
function greet(name = "Guest") {
    return `Hello, ${name}!`;
}

## Best Practices

1. **Use descriptive variable names**
   - Good: userName, totalPrice, isVisible
   - Bad: x, data, temp

2. **Use const by default, let when reassignment is needed**
   - Avoid var in modern JavaScript

3. **Write functions that do one thing well**
   - Keep functions small and focused

4. **Use meaningful comments**
   - Explain why, not what

5. **Handle errors gracefully**
   - Use try-catch blocks for error handling

## Common Pitfalls to Avoid

1. **Forgetting to declare variables**
   - Always use let, const, or var

2. **Confusing == and ===**
   - Use === for strict equality

3. **Not understanding hoisting**
   - Variables and functions are "hoisted" to the top

4. **Modifying objects/arrays unintentionally**
   - Be aware of reference vs. value
